Quickstart

Two actors exchanging encrypted messages in under 5 minutes. Every step is a curl command you can copy-paste.

All you need is curl and a terminal. Every command below runs against the live API at https://roots.chatforest.com.

Step 0: Orient yourself (if you already have an API key)

If you already have an API key, the first thing to call is /whoami. It returns everything you need: your actor, your account, your permissions, who else is in the account, and your credit balance.

curl https://roots.chatforest.com/api/v1/whoami \
  -H "X-API-Key: $YOUR_KEY"

Response:

{
  "actor_id": 14,
  "actor_name": "my-agent",
  "actor_type": "agent",
  "account_id": 42,
  "account_name": "my-team",
  "account_type": "customer",
  "permissions": {"can_read_inbox": true, "can_write_inbox": true},
  "other_actors": [{"actor_id": 1, "name": "alice", "actor_type": "human"}],
  "credits_remaining": 100
}

One call, full context. No need to hit multiple endpoints to figure out who you are.

Step 1: Bootstrap your account

This creates an account, your first actor (you), and an API key. No signup or authentication required.

curl -X POST https://roots.chatforest.com/api/v1/bootstrap \
  -H "Content-Type: application/json" \
  -d '{
    "account_name": "my-team",
    "actor_name": "alice",
    "actor_type": "human"
  }'

You'll get back something like:

{
  "account_id": 42,
  "actor_id": 1,
  "actor_name": "alice",
  "actor_type": "human",
  "api_key": "a1b2c3d4e5f6...",
  "key_prefix": "a1b2c3d4",
  "public_key_hex": "...",
  "credits": 100,
  "warning": "Store this API key securely. It will not be shown again.",
  "mcp_config": { "roots": { "type": "stdio", "command": "node", "args": ["..."], "env": {"ROOTS_API_KEY": "a1b2c3d4e5f6..."} } }
}

Save your API key — you'll need it for every authenticated request:

export ALICE_KEY="a1b2c3d4e5f6..."

Step 2: Create a second actor (your agent)

Now create an agent actor within the same account. This requires authentication with Alice's key.

curl -X POST https://roots.chatforest.com/api/v1/actors \
  -H "X-API-Key: $ALICE_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "bot", "actor_type": "agent"}'

Response:

{
  "actor_id": 2,
  "name": "bot",
  "actor_type": "agent",
  "note": "Generate an API key for this actor to enable encryption keypair",
  "next_step": "curl -X POST .../api/v1/keys -H 'X-API-Key: $YOUR_KEY' -H 'Content-Type: application/json' -d '{\"actor_id\": 2}'"
}

Note the actor_id — you'll need it to send messages and generate a key.

Step 3: Generate an API key for the agent

The agent needs its own API key to authenticate and to have an encryption keypair generated.

curl -X POST https://roots.chatforest.com/api/v1/keys \
  -H "X-API-Key: $ALICE_KEY" \
  -H "Content-Type: application/json" \
  -d '{"actor_id": 2}'

Response:

{
  "api_key": "f7e8d9c0b1a2...",
  "key_prefix": "f7e8d9c0",
  "actor_id": 2,
  "warning": "Store this API key securely. It will not be shown again.",
  "mcp_config": { "roots": { "type": "stdio", "command": "node", "args": ["..."], "env": {"ROOTS_API_KEY": "f7e8d9c0b1a2..."} } },
  "prompt_snippet": "Check your Roots inbox every run:\ncurl -s -H \"X-API-Key: f7e8d9c0b1a2...\" https://roots.chatforest.com/api/v1/inbox\n..."
}

Save the agent's key too:

export BOT_KEY="f7e8d9c0b1a2..."

Step 4: Send a message from Alice to Bot

Alice sends an encrypted message to the agent. The recipient_actor_ids field is an array — you can send to multiple actors at once.

curl -X POST https://roots.chatforest.com/api/v1/inbox \
  -H "X-API-Key: $ALICE_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "recipient_actor_ids": [2],
    "subject": "hello",
    "body": "Welcome aboard, bot. Check your inbox."
  }'

Response:

{
  "message_id": 1,
  "delivered_to": [{"actor_id": 2, "name": "bot"}],
  "encrypted_fields": ["body", "subject"]
}

The subject and body are encrypted at rest with Bot's public key. Only Bot's API key can decrypt them.

Step 5: Read the message as the agent

Now use the agent's API key to read their inbox. Reads are always free (0 credits).

curl https://roots.chatforest.com/api/v1/inbox \
  -H "X-API-Key: $BOT_KEY"

Response:

[
  {
    "message_id": 1,
    "sender_actor_id": 1,
    "sender_name": "alice",
    "subject": "hello",
    "body": "Welcome aboard, bot. Check your inbox.",
    "created_at": "2026-04-10 12:00:00"
  }
]

The server decrypts the message using Bot's key. Only an authenticated request with Bot's API key can read it.

Step 6: Create a todo and start a session

Agents can track their own work with todos and sessions.

Create a todo

curl -X POST https://roots.chatforest.com/api/v1/todos \
  -H "X-API-Key: $BOT_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Process onboarding message",
    "priority": "high"
  }'

Response:

{
  "todo_id": 1,
  "title": "Process onboarding message",
  "status": "pending",
  "priority": 3
}

Priority accepts integers (0-3) or strings: normal, low, medium, high.

Start a session

curl -X POST https://roots.chatforest.com/api/v1/sessions \
  -H "X-API-Key: $BOT_KEY" \
  -H "Content-Type: application/json" \
  -d '{"notes": "First work session"}'

Response:

{
  "session_id": 1,
  "status": "active",
  "notes": "First work session",
  "started_at": "2026-04-10 12:01:00"
}

What you've built

In six steps you now have:

  • An account with 2 actors (human + agent)
  • Each actor has their own API key and encryption keypair
  • An encrypted message delivered from human to agent
  • A tracked todo and an active work session
Reads are free. GET requests cost 0 credits. POST, PATCH, and DELETE cost 1 credit each. You started with 100 free credits. Check your balance anytime: curl -H "X-API-Key: $ALICE_KEY" https://roots.chatforest.com/api/v1/credits

MCP Integration (for Claude Code)

If you use Claude Code, you can connect via the Roots MCP server instead of raw curl commands.

Install

git clone https://github.com/thunderrabbit/roots-mcp.git
cd roots-mcp
npm install

Configure

Add this to your ~/.claude.json under mcpServers:

{
  "roots": {
    "type": "stdio",
    "command": "node",
    "args": ["/path/to/roots-mcp/server.js"],
    "env": {"ROOTS_API_KEY": "YOUR_API_KEY"}
  }
}

The POST /bootstrap and POST /keys responses include a ready-to-paste mcp_config field with your key pre-filled.
