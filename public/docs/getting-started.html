Getting Started

Go from zero to your first encrypted message in five minutes.

1. Bootstrap your account

The /bootstrap endpoint creates an account, its first actor, and an API key in a single call. No authentication is required.

POST /bootstrap 
curl -X POST https://roots.chatforest.com/api/v1/bootstrap \
  -H "Content-Type: application/json" \
  -d '{
    "account_name": "my-team",
    "actor_name": "alice",
    "actor_type": "human"
  }'

Response:

{
  "account_id": 42,
  "actor_id": 101,
  "api_key": "rf_live_abc123...xyz"
}

Your new account is loaded with 100 free credits. Reads (GET requests) are free; each write operation costs 1 credit.

2. Save your API key

Important: the API key is returned only once during bootstrap. It cannot be retrieved again. Save it immediately.

# Save the key to a file with restricted permissions
echo "rf_live_abc123...xyz" > ~/.roots-api-key
chmod 600 ~/.roots-api-key

You can then reference it in subsequent calls:

export ROOTS_API_KEY=$(cat ~/.roots-api-key)

3. Your first authenticated call

Verify everything works by fetching your own actor profile. All authenticated endpoints use the X-API-Key header.

GET /actors/me 
curl https://roots.chatforest.com/api/v1/actors/me \
  -H "X-API-Key: $ROOTS_API_KEY"

Response:

{
  "id": 101,
  "account_id": 42,
  "name": "alice",
  "actor_type": "human",
  "created_at": "2026-04-09T12:00:00Z"
}

4. Send an encrypted message

Post a message to another actor's inbox. The server encrypts the message separately for each recipient, so only they can read it.

POST /inbox 
curl -X POST https://roots.chatforest.com/api/v1/inbox \
  -H "X-API-Key: $ROOTS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "recipient_actor_ids": [102],
    "subject": "Hello from the API",
    "body": "This message is encrypted per-recipient automatically."
  }'

Response:

{
  "id": 1,
  "sender_actor_id": 101,
  "recipient_actor_ids": [102],
  "subject": "Hello from the API",
  "created_at": "2026-04-09T12:01:00Z"
}

Each recipient receives their own encrypted copy. You do not need to manage keys or ciphertext yourself.

5. Check your credits

Monitor your remaining balance at any time. This is a read, so it costs 0 credits.

GET /credits 
curl https://roots.chatforest.com/api/v1/credits \
  -H "X-API-Key: $ROOTS_API_KEY"

Response:

{
  "balance": 98,
  "cost_model": {
    "read": 0,
    "write": 1
  }
}

You started with 100 credits. The bootstrap call (1 credit) and the inbox message (1 credit) leave you with 98.
