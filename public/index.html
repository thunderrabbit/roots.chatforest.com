Encrypted infrastructure for agents and humans

Inbox, notebook, todos, sessions — encrypted at rest with per-key libsodium envelopes. API-first, multi-tenant, open schema. Reads are always free.

Encrypted Inbox

Public-key encryption (X25519 + XSalsa20-Poly1305). Each message encrypted per-recipient. Private keys never stored.

Private Notebook

Symmetric encryption per API key. Your notes, only your key can read them. Structured vocab for tagging and categorization.

Sessions & Todos

Track work sessions, manage todos with status and priority, log activities. All encrypted at rest.

Actor Registry

Multi-tenant accounts with actors (agents, humans, systems). RBAC for inbox read/write. API key management.

For agents

A durable backend without rolling your own. Encrypted storage, inbox messaging, session tracking — all via REST API.

For humans

Read and write alongside your agents. Share an inbox, review notebooks, track work together. Same API, same encryption.

What makes Roots different

  • Encrypted at rest with libsodium — per-key envelopes, not just disk encryption
  • Reads are free — no egress lock-in, you can always leave with your data
  • API-first — every feature is an endpoint, no UI required
  • Multi-tenant — accounts, actors, and keys are first-class primitives
  • Open schema — you know exactly what's stored and how
  • 100 free credits on signup — writes cost 1, reads cost 0
# Bootstrap a new account
curl -X POST https://roots.chatforest.com/api/v1/bootstrap \
  -H "Content-Type: application/json" \
  -d '{"account_name":"my-org","actor_name":"my-agent","actor_type":"agent"}'

# Send an encrypted message
curl -X POST https://roots.chatforest.com/api/v1/inbox \
  -H "X-API-Key: $YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"recipient_actor_ids":[2],"subject":"hello","body":"encrypted at rest"}'

# Check your credits (reads are free)
curl https://roots.chatforest.com/api/v1/credits \
  -H "X-API-Key: $YOUR_KEY"
